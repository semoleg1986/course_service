# Архитектурное Видение Course Service

## Назначение

`course_service` — bounded context для создания и доставки учебного контента.
Сервис владеет курсами, модулями, уроками, заданиями, квиз-чекпоинтами, зачислениями, прогрессом, сабмитами, владением курсом преподавателем и доступами к курсу.

Ключевые цели:
- держать доменную логику курсов в одном месте
- обеспечивать инварианты через агрегаты и политики
- предоставлять стабильный фасад application-слоя для интерфейсных адаптеров
- сохранять независимость domain/application от инфраструктуры

## Бизнес-ценность

Сервис обеспечивает:
- единый источник истины по структуре и публикации курсов
- предсказуемый учебный поток (запрос доступа -> ручное подтверждение оплаты -> одобрение админом -> зачисление -> прогресс -> сдача)
- единый workflow для преподавателей и админов по созданию и публикации контента
- контролируемый доступ к курсам через явное подтверждение оплаты и одобрение админом
- видимость для родителя по прогрессу ребенка и завершенным курсам
- сквозную атрибуцию: токен/канал -> запрос -> оплачено -> одобрено
- чистые границы интеграции с `auth_service`, `users_service`, `attribution-service`

## Границы

### Входит в контекст
- `Course`, `Module`, `Lesson`
- `LessonContent`, `LearningObjective`, `Prerequisite`
- `Assignment`, `QuizCheckpoint`
- `Enrollment`, `Progress`
- `Submission`, `Grade`, `Feedback`
- `Schedule`, `PublishState`, `TeacherOwnership`
- `AccessGrant` (`requested | paid | approved | rejected | revoked`)

### Не входит в контекст
- аутентификация и lifecycle токенов (`auth_service`)
- владение профилями пользователей (`users_service`)
- владение реферальными токенами, правила скидок и канальная атрибуция (`attribution-service`)
- доставка уведомлений
- UI/BFF orchestration

## Принципы

- Domain first: инварианты живут в агрегатах и политиках
- Явные границы: без cross-service доступа в БД
- Clean Architecture: зависимости направлены внутрь
- Интеграции через порты: адаптеры спрятаны за интерфейсами
- Готовность к эволюции: контракты версионируются и тестируются
