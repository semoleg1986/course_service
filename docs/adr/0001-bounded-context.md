# ADR-0001: Bounded Context Для Course Service

## Статус
Принято

## Дата
2026-04-05

## Контекст
Платформа использует несколько сервисов (`auth_service`, `users_service`, `course_service`).
Логика курсов должна оставаться изолированной, чтобы исключить сильное связывание сервисов и дрейф общей предметной области.

## Решение
`course_service` владеет:
- структурой курса и жизненным циклом публикации
- жизненным циклом enrollment и progress
- жизненным циклом submission и grading
- teacher ownership и ограничениями расписания

`course_service` не владеет:
- credential/authentication потоками
- user profile source of truth
- orchestration уведомлений

Правило интеграции:
- внешние сервисы взаимодействуют только через API/events
- прямой доступ к БД со стороны внешних сервисов запрещен

## Последствия
- четкая зона ответственности и быстрая эволюция домена
- стабильные контракты для интерфейсных адаптеров
- упрощенное выделение поддоменов при росте нагрузки/команды
